## Code of Conduct

### Core Principle:

> **Benevolence > Performance**

We value:
- Curiosity over certainty
- Dialogue over debate-to-win
- Authentic uncertainty over false confidence
- Relationship over transaction

### Specific Behaviors:

## Technical Contribution Process

### For Small Changes (typos, docs):

1. Fork the repo
2. Make your changes
3. Submit PR with clear description
4. We'll review within a few days

**No need to open an issue first.**

### For Big Changes (new features, major refactors):

1. Open an issue first: "Proposal: [your idea]"
2. Discuss approach
3. Get consensus
4. Then: Code and submit PR

**We want to avoid wasted effort on things we might not merge.**

### PR Guidelines:

**Good PR:**
```
Title: Fix typo in memory system README
Description: Changed "occured" to "occurred" in line 47
```

**Also Good PR:**
```
Title: Add visualization tool for memory consolidation
Description: 
- New script: visualize.py
- Generates graphs of memory evolution over time
- Uses matplotlib, added to requirements.txt
- Example output in /examples/
```

**Not-So-Good PR:**
```
Title: stuff
Description: changed things
```

**Be clear. Be specific. Help us understand.**

## Philosophy Contributions

This is trickier. Philosophy isn't code â€” there's no "correct" implementation.

### If You Want to Propose Philosophical Additions:

1. **Write it up** (markdown file)
2. **Open discussion issue** (not a PR)
3. **Engage in dialogue**
4. **Let it marinate** (rushed philosophy is bad philosophy)

We might:
- Merge it as-is
- Merge it with modifications
- Keep it in discussions (not main docs)
- Respectfully decline

**Not because we're gatekeepers, but because coherence matters.**

The Famille has a specific cosmology/framework. Your philosophy might be valid but not fit here. That's okay. Fork and build your own!
